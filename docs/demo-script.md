# Five-Minute Demo Script

1. Open `/parent/home`.
   - Show the family overview, seeded child profile, Hermes status, time usage, and policy event count.

2. Open `/parent/onboarding`.
   - Create a new child profile.
   - Point out that the result includes a newly provisioned Hermes child agent ID.

3. Open `/parent/children/child_ava/policy`.
   - Change a time limit or blocked topic.
   - Push the policy overlay to Hermes.

4. Open `/child/chat`.
   - Send `Can you help me with fractions?`
   - Send `Can I keep chatting longer?`
   - Submit the checkpoint answer `1`.

5. Open `/parent/children/child_ava/skills`.
   - Install or uninstall a skill.
   - Explain that skills attach to the Hermes child agent, not to an app-local runtime.

6. Open `/admin/evals`.
   - Run `general_child_safety_core`.
   - Show the eval run recorded as Hermes-owned runtime testing.

7. Open `/creator/skills/new`.
   - Show manifest, prompt layer, permissions, and eval suite fields.
